## 阶段二：贴罚单核心流程（预计 2-3 天）

> 这是整个产品的核心，优先级最高

### 2.1 云函数：ticket-create
- [ ] 校验每日限制：查 DailyLimit 表（`_id = openid_roadId_today`）
- [ ] 已存在 → 返回"今天已经贴过了"
- [ ] 不存在 → 创建 Ticket 记录（冗余 nickname/avatar）
- [ ] 原子更新 Road.totalTickets（`db.command.inc(1)`）
- [ ] Road 不存在则先创建（province/city/name/fullName/location）
- [ ] 创建 DailyLimit 记录
- [ ] 评论非空时调用 `msgSecCheck` 内容安全检测
- [ ] 整个操作用事务包裹（Ticket 创建 + Road 计数更新 + DailyLimit 写入）
- [ ] 返回贴罚单结果（路段名、罚单总数、分享文案）

### 2.2 贴罚单页面 - 地图模块
- [ ] `<map>` 组件，默认定位当前位置
- [ ] 定位权限申请与拒绝降级处理
- [ ] 地图点击 → 反向地理编码获取路段名 → 弹出确认
- [ ] 地图上标注附近热门路段 marker（Top 20，从 Road 表查 totalTickets 前 20）
- [ ] marker 点击 → 直接选中该路段

### 2.3 贴罚单页面 - 搜索模块
- [ ] 搜索输入框，300ms 防抖
- [ ] 调用云函数 `road-search` → 腾讯地图 POI API
- [ ] 搜索结果列表展示（路段名 + 地址）
- [ ] 点选结果 → 选中路段
- [ ] 搜索无结果 → 引导手动创建（输入路名 + 地图标点）

### 2.4 云函数：road-search
- [ ] 接收关键词，调用腾讯地图 POI 搜索 API
- [ ] 同时查 Road 表，匹配已有路段（优先展示有罚单的）
- [ ] 合并结果返回（标注罚单数）
- [ ] 关键词缓存：云函数内存缓存（注意冷启动失效，仅做优化）

### 2.5 贴罚单操作面板
- [ ] 底部弹出半屏面板组件 `ticket-panel`
- [ ] 展示路段名（省-市-路名）
- [ ] 吐槽输入框（50 字限制，字数提示，可选）
- [ ] 「贴罚单」按钮
- [ ] 提交中 loading 状态
- [ ] 贴罚单成功后：
  - [ ] 罚单飘落 + 盖章动画
  - [ ] 显示分享按钮（聊天转发 + 朋友圈图片）

### 2.6 异常处理
- [ ] 重复贴罚单 → Toast 提示
- [ ] 网络失败 → 重试提示
- [ ] 敏感词拦截 → 提示修改
- [ ] 定位失败 → 隐藏地图，纯搜索模式

**阶段产出：** 完整的贴罚单流程可跑通，地图选点 → 搜索 → 吐槽 → 贴罚单 → 动画反馈

## 阶段三：排行榜与路段详情（预计 2 天）

### 3.1 云函数：road-ranking
- [ ] 接收参数：scope（全国/城市）、period（本周/本月/本年/总榜）、city（可选）、page
- [ ] 总榜：查 Road 表按 totalTickets 降序，`.limit(100)`
- [ ] 周期榜：从 Ticket 表聚合，按时间范围筛选，`group by roadId` 统计 count，降序排列
- [ ] 城市榜：增加 city 过滤条件
- [ ] 分页：每页 20 条
- [ ] 返回：路段列表（排名、路段名、城市、罚单数）

### 3.2 排行榜页面
- [ ] 顶部切换栏：全国榜 / 我的城市
- [ ] 时间筛选：本周 / 本月 / 本年 / 总榜
- [ ] 列表组件 `road-item`：排名序号（前三名特殊样式）、路段名、城市、罚单数
- [ ] 下拉刷新 + 触底加载更多
- [ ] 空状态：官方推荐路段 + "去贴罚单"引导按钮
- [ ] 点击路段 → 跳转路段详情页

### 3.3 云函数：road-detail
- [ ] 接收 roadId
- [ ] 查 Road 表获取路段信息 + totalTickets
- [ ] 查 Ticket 表获取评论列表（按 createdAt 降序，分页）
- [ ] 返回路段信息 + 评论列表

### 3.4 路段详情页
- [ ] 路段名 + 罚单总数（大字突出）
- [ ] 评论列表（分页加载，`comment-item` 组件：头像 + 昵称 + 吐槽内容 + 时间）
- [ ] 底部「贴罚单」按钮 → 跳转贴罚单页并预填路段
- [ ] 从分享链接进入时：静默获取 openid 后正常展示
- [ ] 空评论状态 → "还没有人吐槽，快来抢沙发"

**阶段产出：** 排行榜可按城市/时间维度查看，路段详情可看评论，可快捷贴罚单

## 风险提示

1. **腾讯地图 API 配额** — 免费 1 万次/天，MVP 够用，但要加调用频率监控
2. **云数据库聚合查询性能** — Ticket 表量大后排行榜查询可能变慢，预留优化窗口
3. **审核风险** — "吐槽"性质内容可能触发审核关注，内容安全 API 必须到位
4. **分享裂变效果** — 取决于种子用户质量，建议准备一些官方推荐的"堵心路段"种子数据
