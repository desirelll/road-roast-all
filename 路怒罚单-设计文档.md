# 路路辣评 (Road Roast) - 微信小程序设计文档

## 1. 产品概述

**名称**：路路辣评 (Road Roast)
**定位**：城市通勤路况吐槽平台，轻社交型
**一句话描述**：给最堵心的路段贴罚单，看全国最差路段排行榜

**核心交互**：用户在地图上定位/搜索路段 → 给它贴罚单 → 罚单越多的路排名越高

## 2. 功能范围（MVP）

### 包含
- 贴罚单：地图定位 + 搜索路段，每天每用户每条路限 1 次
- 排行榜：全国榜 / 城市榜，支持本周 / 本月 / 本年 / 总榜
- 评论吐槽：贴罚单时可附一句吐槽（50 字，可选），路段详情页展示评论
- 分享裂变：聊天转发卡片 + 朋友圈分享图片（Canvas 绘制）
- 路段详情：罚单总数、评论列表
- 个人中心：罚单记录、评论记录、统计

### 不包含（后续版本）
- 用户关注/粉丝系统
- 路段图片上传
- 推送通知
- 多语言

## 3. 页面结构

三个 Tab 页，底部 TabBar 导航。

### Tab 1：贴罚单
- 顶部搜索框（搜索路段名，调用腾讯地图 POI API）
- 中间地图组件（微信内置 map），定位当前位置，地图上标注附近热门路段（Top 20）
- 路段选择流程（优先级从高到低）：
  1. 用户在搜索框输入关键词 → POI 搜索结果列表 → 点选
  2. 用户点击地图上某个位置 → 反向地理编码得到路段名 → 确认
  3. 以上都不满意 → 手动输入路名 + 地图标点 → 创建新路
- 选定路段后，底部弹出操作面板：
  - 路段名（省-市-路名）
  - 文字输入框（吐槽，可选，50 字限制）
  - 「贴罚单」按钮
- 贴完后：罚单动画反馈（飘落 + 盖章效果） → 显示分享按钮

### Tab 2：排行榜
- 顶部切换：全国榜 / 我的城市
- 时间筛选：本周 / 本月 / 本年 / 总榜
- 列表项：排名序号 | 路段名 | 罚单数
- 点击路段 → 路段详情页（罚单总数、评论列表）
- 空状态：官方推荐路段 + 引导去贴罚单

### Tab 3：我的
- 头像 + 昵称（按需微信授权）
- 我的罚单记录（时间线列表）
- 我的评论
- 贴罚单总数统计

### 路段详情页（非 Tab，从排行榜/分享进入）
- 路段名 + 罚单总数
- 评论列表（分页加载）
- 「贴罚单」快捷入口

## 4. 数据模型

### User
| 字段 | 类型 | 说明 |
|------|------|------|
| _id | String | 自动生成 |
| openid | String | 微信 openid |
| nickname | String | 昵称 |
| avatar | String | 头像 URL |
| totalTickets | Number | 累计贴罚单数 |
| createdAt | Date | 注册时间 |

### Road
| 字段 | 类型 | 说明 |
|------|------|------|
| _id | String | 自动生成 |
| province | String | 省份 |
| city | String | 城市 |
| name | String | 路段名 |
| fullName | String | 完整名称（省-市-路名） |
| location | GeoPoint | 经纬度 |
| totalTickets | Number | 累计罚单数 |
| createdAt | Date | 创建时间 |

### Ticket
| 字段 | 类型 | 说明 |
|------|------|------|
| _id | String | 自动生成 |
| userId | String | 用户 ID |
| roadId | String | 路段 ID |
| comment | String | 吐槽内容（可选） |
| nickname | String | 冗余存储用户昵称（用户改名时不同步历史） |
| avatar | String | 冗余存储用户头像 URL |
| createdAt | Date | 贴罚单时间 |

**索引：**
- `roadId + createdAt` 复合索引（排行榜聚合）
- `userId + createdAt` 复合索引（个人记录查询）

### DailyLimit
| 字段 | 类型 | 说明 |
|------|------|------|
| _id | String | `userId_roadId_YYYY-MM-DD`（模拟唯一约束） |
| userId | String | 用户 ID |
| roadId | String | 路段 ID |
| date | String | 日期字符串（YYYY-MM-DD） |

**清理策略**：云函数定时触发器每天清理 7 天前的记录。

## 5. 技术架构

### 前端
- 微信小程序原生开发
- 页面：index（贴罚单）、ranking（排行榜）、profile（我的）、road-detail（路段详情）

### 后端：微信云开发

**云数据库**：4 张表（User / Road / Ticket / DailyLimit）

**云函数**（6 个）：

| 函数 | 职责 |
|------|------|
| ticket-create | 贴罚单：校验每日限制、创建 Ticket、原子更新 Road.totalTickets |
| ticket-list | 查询我的罚单记录（分页） |
| road-search | 搜索路段（调用腾讯地图 POI API），云函数层关键词缓存 5 分钟 |
| road-ranking | 排行榜查询（全国/城市），从 Ticket 表按时间范围聚合，建好索引直接查 |
| road-detail | 路段详情 + 评论列表（分页） |
| user-info | 获取/更新用户信息 |

**云存储**：分享卡片图片、小程序码

### 关键技术决策

1. **原子计数**：Road.totalTickets 使用 `db.command.inc(1)` 原子操作
2. **懒重置（周期计数）**：排行榜查询时根据时间范围从 Ticket 表聚合计算（本周/本月/本年），不依赖定时器批量重置 Road 表字段。Road 表只保留 totalTickets 一个累计字段
3. **排行榜查询**：砍掉云函数内存缓存，依靠 Ticket 表复合索引 + `.limit(100)` 直接聚合查询
4. **路段数据**：用户搜索时调用腾讯地图 POI 搜索，前端 300ms 防抖，选定后查 Road 表是否存在，不存在则新建
5. **静默登录**：小程序启动时通过云开发自动获取 openid，无需用户主动授权
6. **评论展示**：Ticket 表冗余存储 nickname/avatar，避免评论列表查询时 join User 表
7. **评论安全**：调用微信内容安全 API `msgSecCheck` 过滤敏感词
8. **数据校验**：云函数中通过 openid 校验身份，不信任前端传入的 userId
9. **每日限制**：DailyLimit 表用 `_id = userId_roadId_date` 模拟复合唯一约束
10. **小程序码**：用云调用 `wxacode.getUnlimited` 生成，按路段缓存到云存储

## 6. 分享裂变

### 聊天转发
- 贴罚单成功后：标题 "我给XX路贴了一张罚单，它已累计X张"，路径指向路段详情页
- 排行榜页：标题 "来看看你城市的路有多堵"

### 朋友圈分享
- Canvas 绘制分享图片：大字标题 + 罚单数 + 城市名 + 小程序码
- 用户保存到相册后手动发朋友圈

## 7. 异常处理

| 场景 | 处理 |
|------|------|
| 搜索不到路段 | 允许手动创建（输入路名 + 地图标点） |
| 重复贴罚单 | 前端提示 "今天已经给这条路贴过罚单了" |
| 评论含敏感词 | 拦截并提示修改 |
| 贴罚单网络失败 | 提示重试 |
| 路段数据为空 | 排行榜显示官方推荐 + 引导去贴罚单 |
| 定位权限被拒绝 | 贴罚单页隐藏地图，仅显示搜索框，提示开启定位 |
| 用户未授权昵称头像 | 「我的」页显示默认头像 + "微信用户"，按需引导授权 |
| 分享链接打开（未登录） | 静默获取 openid 后正常显示，贴罚单无需昵称头像 |
| 地图加载失败 | 降级为纯搜索模式，隐藏地图区域 |

| 空状态 | 引导去贴罚单 | 加种子数据/官方推荐 |

| 朋友圈图片 | 路段名 + 罚单数 + 小程序码 | 大字标题 + 罚单数 + 城市名，抓眼球 |
